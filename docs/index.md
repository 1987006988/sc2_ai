# Documentation Index

## Foundation

- `foundation/01_survey/`: pre-project survey materials and source summaries.
- `foundation/02_feasibility/`: feasibility and route-comparison materials.
- `foundation/03_direction/`: direction lock-in, phase-1 scope, and project-start summary.

## Context

- `context/`: current project consensus, live status, roadmap, glossary, system map, and validated findings.

## Decisions

- `adr/`: architecture decision records.

## Process

- `plans/`: active, completed, and abandoned task plans.
- `handoffs/`: session handoffs and recovery notes.
- `lessons/`: validated conclusions and recurring pitfalls.
- `templates/`: reusable task, plan, ADR, experiment, and handoff templates.
- `commands/`: common commands and verification matrix.

## Why Foundation Is Separate

`foundation/` stores the historical inputs that justified the project:

- survey report;
- feasibility report;
- direction and scope lock-in.

These files should remain mostly read-only so later status updates do not blur the original rationale.

`context/` stores the current living consensus and is expected to change over time.
