# Opponent Modeling Research

Prototype area for opening classification, hidden-state inference, rush risk, tech risk, and enemy army estimation.

This is the phase-1 priority research line, but code here is not mainline.

## Promotion Path

1. Define features and labels.
2. Run repeatable experiments.
3. Write a report in `reports/`.
4. Create a promotion plan in `docs/plans/active/`.
5. Port only the stable interface or implementation into `src/sc2bot/opponent_model/`.

## Phase-1 Prototype Deliverables

- input field definition;
- output field definition;
- with/without opponent model ablation protocol;
- rule-based baseline design.
